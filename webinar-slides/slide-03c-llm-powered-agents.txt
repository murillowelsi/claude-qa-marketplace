================================================================================
  SLIDE 3C: LLM-POWERED AGENTS — WHAT CLAUDE CODE ACTUALLY IS

================================================================================

  ISTQB CT-GenAI defines the architecture your skills run on.
  Understanding it explains why the skills behave the way they do.

  THE ARCHITECTURE
  ----------------

  LLM-POWERED AGENT
    An AI system that doesn't just respond — it takes actions.
    It perceives input, reasons about it, and invokes tools
    to interact with external systems.

  SEMI-AUTONOMOUS AGENT
    The agent acts independently within a task, but pauses
    for human confirmation at key decision points.
    This is by design — not a limitation.

  TOOLS
    Predefined functions the agent can invoke.
    Examples: fetch a Jira ticket, navigate a browser,
    write a file, create a bug report, run a test.

  ORCHESTRATION
    The coordination layer that decides which tool to call,
    in what order, based on the task and intermediate results.
    This is what makes a skill a workflow — not just a prompt.

  HOW CLAUDE CODE SKILLS MAP TO THIS MODEL
  -----------------------------------------

  Claude Code = the agent runtime
  SKILL.md    = the system prompt that defines role + instructions
  MCP tools   = the "tools" the agent invokes (Jira, browser, filesystem)
  Skill steps = the orchestration logic
  Human review at the end = the semi-autonomous checkpoint

  EXAMPLE: /explore-and-plan
  --------------------------
  1. Agent receives URL (input)
  2. Invokes browser tool → navigates the app (tool use)
  3. Discovers features and flows (reasoning)
  4. Invokes screenshot tool → captures evidence (tool use)
  5. Synthesizes a full manual test plan (output)
  6. Saves to file and asks: "Should I create a Jira ticket?" (checkpoint)

  This is not a chatbot. This is an LLM-powered agent
  doing structured work with real tools.

  THE HUMAN-IN-THE-LOOP IS A FEATURE
  ------------------------------------
  The tester reviews AI-generated testware before it ships.
  The agent handles the repetitive, structural work.
  The tester applies judgment, context, and domain knowledge.

  That division is what ISTQB calls the "semi-autonomous" model —
  and it is the right model for QA.
