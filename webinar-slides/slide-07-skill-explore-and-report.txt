  /explore-and-plan
  -----------------
  THE PROBLEM: Before a release, teams need a test plan — but no one has
  time to manually walk through the whole app and document every scenario.
  Plans are either skipped or written from memory and full of blind spots.

  WHAT IT DOES:
  The agent opens a real browser, navigates the live application,
  and systematically explores every reachable section.

  It performs:
    - Full app navigation and snapshot analysis
    - Authentication (if credentials are provided)
    - Interaction with forms, buttons, modals, filters, pagination
    - Identification of feature areas, key user flows, and risk zones

  It produces a complete Manual Test Plan covering:
    - Happy path scenarios
    - Negative / error paths
    - Edge cases and permissions
    - Non-functional observations

  Output saved to: qa-output/test-plans/<app>-test-plan-<date>.md
  Optionally posts the plan directly to Jira as a Task.

  COMMAND: /explore-and-plan https://your-app.com

  /report-bug
  -----------
  THE PROBLEM: When a test fails, writing a good bug report takes time.
  Poorly written reports cause developer back-and-forth and slow down fixes.

  WHAT IT DOES:
  Turns a failed test into a structured, developer-ready bug report:
    - One-sentence summary (specific, reproducible)
    - Severity: Critical / High / Medium / Low
    - Priority: Urgent / High / Medium / Low
    - Environment, reproducibility, steps, expected vs actual result
    - Auto-creates the Jira bug ticket
    - Links the bug to the originating story

  Output saved to: qa-output/bug-reports/<slug>-bug-<date>.md

  COMMAND: /report-bug (describe what broke, or it reads your session context)
